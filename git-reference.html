How Git Works

Git tracks your work across four zones. Every command moves changes between them.

📁
Working Directory
Your actual files on disk. Where you edit.
git add
git restore
📋
Staging Area
(Index)
Changes queued for the next commit.
git commit
git reset --soft
🗄️
Local Repository
(.git folder)
Full history stored here, on your machine.
git push
git pull / fetch
☁️
Remote Repository
(GitHub, GitLab…)
Shared with your team online.
📸 A commit is a snapshot
Not just a diff — Git stores a complete picture of all tracked files at that moment. You can jump back to any commit instantly.
🌿 Branches are cheap
A branch is just a pointer to a commit — creating one is instant. Use branches freely for features, experiments, and fixes.
🔒 Local history is safe
Most operations only touch your local .git folder. Nothing reaches GitHub until you run git push.
